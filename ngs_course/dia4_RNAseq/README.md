## Introduction to RNAseq

RNA-seq (RNA sequencing) is a method for quantifying gene expression across the transcriptome.

It involves isolating RNA from biological samples, converting it into complementary DNA (cDNA), and sequencing it using high-throughput technologies.

The resulting reads are aligned to a reference genome or transcriptome, allowing the quantification of gene and transcript abundance as well as the identification of differential expression between conditions.

## Project Background

We will analyze publicly available RNA-seq data from the PRJNA998264 project. This study investigates the molecular mechanisms underlying myelodysplastic syndrome (MDS) using a zebrafish model with loss of function of the *tet2* gene.

Finally, we will explore functional implications through pathway enrichment analysis and identify biological processes potentially affected by the observed transcriptional changes.

## 3. Reads alignment to the reference genome

Once we state that the reads show good quality, we can perform the alignment to the reference genome. 

For RNAseq, the selected aligner must be splice-aware, assuring that we retain reads that align to splicing junctions (i.e. the 5' of the read align to one exon and the 3' end to the next exon of a particular gene). 

Several splice-aware aligners have been developed for RNAseq reads alignment (TopHat, Hisat2, STAR, ...), for the practice we will use `Hisat2`.

For the alignment we will need the sequence of the reference genome (FASTA) and the Hisat2 index. 

### 3.1 Build Hisat2 index

```bash
mkdir /home/curso/dia4/hisat2
cd /home/curso/dia4/hisat2

hisat2-build -p 8 ../reference/Danio_rerio.GRCz11.dna.primary_assembly.fa Drerio
```

### 3.2 Align reads with Hisat2

After building the index, we can proceed with the alignment itself. To avoid the long code names in files that also lack any information, we will skip the common characters in SRR codes and call each read by their unique identifier, in this case, the last 2 numbers of the code. We will also feed the `for` loop with a text file that contains the sample IDs.

Use a text editor to build a file named `samples.txt` containing the sample values for the loop: 61 66 67 55 in `/home/curso/dia4`. Then, run alignment:

```bash
# Map the reads to the reference genome

for sample in `cat ../samples.txt`
do
	hisat2 -x Drerio -q -k 1  -p 24 \
	  -U ../reads/SRR254111${sample}.fastq.gz \
	  -S ${sample}.sam 2>> ${sample}_RF_summarymetrics.txt
	samtools view -S -b ${sample}.sam --threads 8 > ${sample}.bam
	samtools sort ${sample}.bam -o ${sample}_sorted.bam --threads 8
	samtools index ${sample}_sorted.bam
	rm -rf ${sample}.bam ${sample}.sam
done
```

## 4. Check strandness of reads (optional)

Only when we are not sure of the library type used for sequencing, we should check strandness. To do that, we will use `infer_experiment.py` script from RSeQC software package. This script must be fed with a sorted bam file and a reference bed file.

### 4.1 Build bed from annotation file

`gff2bed` command will transform the annotation file in GTF format to BED format:

```bash

```

### 4.2. Check strandness

Strandness will be checked just in one sample, assuming that all reads coming from the same experiment will have the same library type.

`infer_experiment.py` will return values associated to strands. To interpret the output, the following table could be of use: 

| Result type	| Meaning | Library type |
| ---   | --- | --- |
| "1++,1--,2+-,2-+" ~1.0   | Reads in the same strand than gene | fr-secondstrand or reverse-stranded |
| "1+-,1-+,2++,2--" ~1.0   | Reads in the opposite strand than gene | fr-firststrand or forward-stranded |
| Both values ~0.5   | Reads without preferential orientation | unstranded |


```bash
# Check strandness

infer_experiment.py -r ../reference/Danio_rerio.GRCz11.112.bed -i 55_sorted.bam
```

## 5. Quantify gene expression

We will count the number of reads in each gene by summing up all reads that align to the annotated exons for that gene with `featureCounts` software. Reads that map to more than one feature will be considered "ambiguous" (CountMultiOverlap is FALSE by default). The flag `-t exon` indicates the feature to be counted. The flag `-g gene_id` will aggregate the counts by gene.


```bash
# Create a directory to store library QC plots and statistics
mkdir /home/curso/dia4/counts
cd /home/curso/dia4/counts

# Count reads by gene
for sample in `cat ../samples.txt`
do
	featureCounts -T 5 -t exon -g gene_id \
		-a ../reference/Danio_rerio.GRCz11.112.gtf \
		-o ${sample}_ftC.tsv ../hisat2/${sample}_sorted.bam 
done
```

## 6. Differential Expression Analysis

Differential expressed genes will be detected with **DESeq2 R package**. As R programming is far beyond the scope of this course, we provide a ready to run R script that outputs the differential analysis results as well as relevant related plots.

We will format the `featureCounts` output to retain only the gene IDs and their counts in each sample, prepare a configuration file that contains the name of each sample, its raw counts file and the condition it belongs to. Then, we will run `deseq2.R` script.

### 6.1 Input counts file formatting

`featureCounts` output includes a .tsv file with counts and additional information. We will retain only the gene ID and count number, without headers, with the following code:

```bash

# Create a folder for differential expression analysis
mkdir /home/curso/dia4/deseq2

# Extract differential expression relevant information and store in a new file: ID in column1 and counts in column7
for sample in `cat ../samples.txt`
do
	    awk -F'\t' '{print $1 "\t" $7}' ${sample}_ftC.tsv | tail -n+3 > ../deseq2/${sample}.tsv
done
```
